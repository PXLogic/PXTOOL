## 1 概述

### 1.1 产品简介

PXTOOL 是一款跨平台桌面应用，用于信号采集、波形显示、测量、搜索、协议解码和数据导出。软件基于 Qt 图形界面，支持 DreamSourceLab 相关仪器，也可打开已保存的数据文件或使用内置演示数据进行查看与分析。

PXTOOL 的主要功能包括：

| 功能 | 说明 |
| --- | --- |
| 设备连接与采集控制 | 选择硬件、演示设备或数据文件，设置采样率、采样深度和采集模式 |
| 数字波形分析 | 逻辑通道显示、触发、搜索、测量、协议解码和毛刺过滤 |
| 模拟/示波器分析 | 模拟波形显示、触发、自动测量、光标测量、FFT 和数学通道 |
| 数据采集 | 连续或离散模拟数据查看、测量、保存与导出 |
| 文件与会话管理 | 打开、保存、导出数据，保存/加载设备配置，会话恢复和截图 |
| 界面配置 | 深色/浅色主题、多语言、显示选项、快捷键和日志设置 |

### 1.2 工作模式

PXTOOL 按设备能力提供不同工作模式。可用模式取决于当前连接的设备或打开的数据文件。

| 模式 | 主要用途 |
| --- | --- |
| 逻辑分析仪 | 采集和分析数字信号，支持触发、模式搜索、协议解码和毛刺过滤 |
| 示波器 | 采集和观察模拟波形，支持触发、自动测量、FFT 和数学运算 |
| 数据采集 | 采集低速或连续模拟数据，适合趋势观察、光标测量和数据导出 |

### 1.3 设备来源

设备选择区域会显示当前可用来源：

| 来源 | 说明 |
| --- | --- |
| 硬件设备 | 通过 USB 连接的真实采集设备 |
| 演示设备 | 用于体验界面和分析流程，无需连接硬件 |
| 文件设备 | 打开已保存的数据文件后，软件以文件内容作为数据来源 |

高采样率或大深度采集会占用较多传输和存储资源。若采集异常，请降低采样率、减少通道数或缩短采样深度。

## 2 主界面

### 2.1 界面组成

PXTOOL 主界面由以下区域组成：

| 区域 | 功能 |
| --- | --- |
| 顶部菜单栏 | 文件、窗口、帮助、主题、语言、显示选项等 |
| 采集工具栏 | 设备选择、采样率、缓存/采样深度、采集模式、开始/停止和单次采集 |

## 3 通用操作

### 3.1 连接设备

1. 将采集设备连接到电脑。
2. 在 Device 列表中选择目标设备。
3. 选择工作模式，例如逻辑分析仪、示波器或数据采集。
4. 打开 Options 面板，确认通道、模式和设备参数。
5. 设置采样率、Buffer 和采集模式。
6. 根据需要配置触发、测量或协议解码。
7. 点击 Start 开始采集。

### 3.2 使用演示设备

在没有硬件时，可选择演示设备查看软件功能。演示设备可用于熟悉波形导航、测量、搜索和协议解码等流程。演示数据不代表真实硬件输入。

### 3.3 打开数据文件

通过 File 菜单的 Open 功能可以打开已保存的数据文件。文件打开后，PXTOOL 会以文件设备方式显示数据，并允许继续进行波形查看、测量、搜索、解码结果查看或导出等操作。

### 3.4 波形导航

波形视图支持常用导航操作：

| 操作 | 说明 |
| --- | --- |
| 左右平移 | 查看采集数据的不同时间位置 |
| 缩放 | 放大查看细节，缩小查看全局 |
| 适配视图 | 将采集数据缩放到合适范围 |
| 跳转到零点 | 回到时间零点或触发附近位置 |
| 光标跳转 | 在测量面板或光标列表中跳转到指定光标 |
| 解码/搜索结果跳转 | 点击结果行可跳转到对应波形位置 |

### 3.5 通道显示调整

在波形左侧通道区域可进行通道显示调整。不同信号类型支持的选项不同。

| 操作 | 说明 |
| --- | --- |
| 启用/禁用通道 | 在 Options 面板或通道自身控件中切换 |
| 修改颜色 | 在通道区域打开颜色选择，设置通道颜色 |
| 修改名称 | 对支持命名的通道设置自定义名称 |
| 调整高度 | 重置单行高度、重置全部高度，或设置为自动、1X、2X、4X、8X |
| 移动通道 | 拖动通道改变显示顺序 |

### 3.6 采集模式

| 模式 | 说明 |
| --- | --- |
| Single | 执行一次采集，完成后停止 |
| Repetitive | 重复采集，适合持续观察稳定信号 |
| Loop | 循环采集，适合长时间观察；部分后处理功能可能不可用 |

### 3.7 Buffer 与磁盘缓存

Buffer 控制一次采集保留的数据量。对于大深度采集，PXTOOL 支持磁盘缓存：

| 设置 | 说明 |
| --- | --- |
| Enable disk cache | 启用磁盘缓存以扩展可采集深度 |
| RAM hot window | 设置保留在内存中的热数据窗口 |
| Disk total depth | 设置磁盘缓存总深度，可选择无限制 |
| Cache directory | 选择缓存目录 |

磁盘缓存适合长时间或大深度采集。缓存目录应选择空间充足、写入速度较高的位置。

## 4 逻辑分析仪

### 4.1 硬件连接

逻辑分析仪模式用于数字信号采集。连接前请确认：

1. 设备已通过 USB 正确连接。
2. 探头地线与被测系统共地。
3. 逻辑电平阈值与被测信号电平匹配。
4. 已启用需要采集的通道。
5. 采样率至少高于目标信号频率的数倍；对协议分析建议留出足够裕量。

### 4.2 设备选项

打开 Options 面板可配置逻辑分析仪的设备参数。具体项目随设备型号和当前模式变化。

常见选项包括：

| 选项 | 说明 |
| --- | --- |
| Operation Mode | 选择设备工作方式，例如普通采集、流式采集或设备支持的其他模式 |
| Channel Mode | 选择通道组合方式；高通道数和高采样率通常不能同时达到最大值 |
| Enable All / Disable All | 快速启用或禁用全部通道 |
| Channel Enable | 单独启用或禁用通道 |
| Threshold Voltage | 设置数字电平判定阈值 |
| Filter Targets | 选择需要应用硬件或设备侧过滤的目标 |
| Max Height | 设置波形最大显示高度 |
| RLE Compress | 启用游程压缩以提升长时间低变化信号的存储效率 |
| External Clock | 使用外部时钟作为采样时钟 |
| Clock Negedge | 使用外部时钟下降沿采样 |
| Stream Buffer | 设置流式采集缓存 |
| Disk Cache | 启用磁盘缓存，扩展大深度采集能力 |

在 Stream 模式下，部分触发和后处理功能会受限，界面会自动禁用不可用选项。

### 4.3 采样率与采样深度

Sample Rate 设置采样频率，Buffer 设置采样深度或时长。采样率越高，时间分辨率越高；采样深度越大，可观察的时间范围越长。

设置建议：

| 场景 | 建议 |
| --- | --- |
| 查看边沿或窄脉冲 | 提高采样率 |
| 分析长时间通信过程 | 增加 Buffer 或启用磁盘缓存 |
| 采集带宽不足 | 降低采样率、减少通道数或缩短采样深度 |
| 内存不足 | 减小 Buffer 或启用磁盘缓存 |

### 4.4 触发

逻辑分析仪提供简单触发和高级触发。

#### 4.4.1 触发位置

Trigger Position 用百分比表示触发点在采集数据中的位置。数值越小，触发前数据越少；数值越大，触发前数据越多。Stream 模式下触发位置可能被限制。

#### 4.4.2 简单触发

Simple Trigger 适合快速设置基础触发条件。可在通道上设置电平或边沿条件，用于捕获目标事件。

触发条件符号含义：

| 符号 | 含义 |
| --- | --- |
| X | 不关心 |
| 0 | 低电平 |
| 1 | 高电平 |
| R | 上升沿 |
| F | 下降沿 |
| C | 上升沿或下降沿 |

#### 4.4.3 高级触发

Advanced Trigger 支持多级触发和串行触发。是否可用取决于设备能力；在 Stream 模式下高级触发不可用。

高级触发包含：

| 类型 | 说明 |
| --- | --- |
| Stage Trigger | 多阶段触发，每个阶段可配置通道条件、反相、计数器、连续条件和逻辑关系 |
| Serial Trigger | 按起始标志、停止标志、时钟标志、数据通道和数据值设置串行触发 |
| Hex Input | 对串行触发数据使用十六进制方式输入 |

### 4.5 采集

设置完成后点击 Start 开始采集。采集中状态区会显示等待触发、已触发、采集进度或已完成等状态。

| 状态 | 说明 |
| --- | --- |
| Waiting for Trigger | 已开始采集，正在等待触发条件 |
| Triggered | 已满足触发条件，继续采集剩余数据 |
| Capturing | 正在采集 |
| Samples Captured | 采集完成并显示捕获样本数 |

若出现 Data Overflow，表示当前采样设置超过了设备或传输能力，应降低采样率、减少通道或缩短采样深度。

### 4.6 搜索

Search 面板用于在逻辑波形中查找指定通道模式。

使用流程：

1. 打开 Search 面板。
2. 展开 Edit per-channel。
3. 在每个通道输入搜索条件。
4. 点击 Search All 生成结果列表。
5. 使用上一项、下一项按钮或点击结果表格跳转。

搜索结果表包含序号、时间和样本位置。搜索符号与触发符号一致，支持 X、0、1、R、F、C。

### 4.7 测量

Measure 面板提供以下测量方式：

| 测量 | 说明 |
| --- | --- |
| Mouse measurement | 鼠标悬停或移动时显示宽度、周期、频率和占空比 |
| Enable floating measurement | 开启或关闭浮动测量提示 |
| Cursor Distance | 添加两点光标，测量时间差和样本差 |
| Edges | 统计指定通道区间内的上升沿、下降沿和总边沿数 |
| Cursors | 管理已创建光标并跳转到对应位置 |

### 4.8 光标

在波形视图中可添加 X 光标或 Y 光标。逻辑分析仪主要使用 X 光标进行时间测量。

常用操作：

| 操作 | 说明 |
| --- | --- |
| 添加光标 | 在 Measure 面板添加，或在波形区域右键添加 |
| 移动光标 | 拖动光标到目标位置 |
| 光标跳转 | 在测量面板中跳转到指定光标 |
| 删除光标 | 使用测量行中的删除按钮 |

### 4.9 协议解码

Decode 面板用于添加和管理协议解码器。PXTOOL 集成 sigrok 解码器，并支持部分 C 解码器。带有 C 实现的协议会在列表中显示 C 和 Py 两种引擎选项。

#### 4.9.1 添加解码器

1. 打开 Decode 面板。
2. 使用搜索框查找协议。
3. 选择 Both、C 或 Py 过滤解码器来源。
4. 点击添加按钮添加协议。
5. 在解码器设置中绑定通道并配置协议参数。

若解码器所需通道未指定，或绑定到了未启用通道，解码结果会显示错误提示。

#### 4.9.2 堆叠解码

部分协议可基于另一个解码器结果继续解码。添加上层解码器后，在设置中选择基础解码器并配置参数即可形成堆叠解码。

#### 4.9.3 协议列表查看器

Protocol List Viewer 以表格方式显示解码注释。可以：

| 操作 | 说明 |
| --- | --- |
| 设置显示内容 | 选择需要显示的协议或字段 |
| 搜索注释 | 在搜索框输入关键字，查看匹配数量 |
| 上一项/下一项 | 在匹配结果间跳转 |
| 导航同步 | 选中表格结果后跳转到对应波形位置 |
| 导出结果 | 将协议列表导出为 CSV 或 TXT |

#### 4.9.4 删除解码器

可删除单个解码器，也可一次删除全部解码器。禁用通道后，PXTOOL 会删除绑定到被禁用通道的相关解码器，以避免错误配置。

### 4.10 毛刺过滤

Filter 面板用于对已采集的逻辑数据执行后处理。

| 功能 | 说明 |
| --- | --- |
| Invert | 翻转整个通道的逻辑极性 |
| Enable | 启用该通道的毛刺移除 |
| Threshold | 设置毛刺脉冲最大宽度，单位为样本点 |
| Mode | 选择 Both、High only 或 Low only，限制要移除的短脉冲类型 |
| Apply | 将当前设置应用到采集数据 |
| Clear | 恢复原始数据 |

毛刺过滤仅对已有逻辑采集数据生效。采集中不能应用；Loop 采集结果不支持该过滤。应用后状态区会显示已处理区域数量或无毛刺等结果。

### 4.11 逻辑分析仪文件操作

逻辑模式支持：

| 操作 | 说明 |
| --- | --- |
| Save | 保存采集数据为 DSView 数据文件 |
| Open | 打开 DSView 数据文件 |
| Export | 导出为支持的外部数据格式 |
| Capture | 保存当前界面截图 |
| Config Load/Store | 加载或保存设备配置文件 |
| Default | 加载当前设备和模式的默认配置 |

## 5 示波器

### 5.1 硬件连接

示波器模式用于观察模拟电压波形。连接前请确认：

1. 探头接地可靠。
2. 输入电压未超过设备允许范围。
3. 探头倍率、耦合方式和垂直档位设置正确。
4. 若需要精确测量，先进行自动校准或手动校准。

### 5.2 设备选项

Options 面板中可配置示波器设备参数。具体可用项随设备型号变化。

常见选项包括：

| 选项 | 说明 |
| --- | --- |
| Operation Mode | 设置示波器工作模式 |
| Bandwidth Limit | 启用或关闭带宽限制 |
| Channel Enable | 启用或关闭模拟通道 |
| Coupling | 选择 GND、DC 或 AC 耦合 |
| V/div | 设置垂直档位 |
| Offset | 调整垂直偏移 |
| Probe Factor | 设置探头倍率 |
| Auto Calibration | 自动校准，校准时不要连接探头 |
| Manual Calibration | 手动调整增益、偏移和补偿参数 |

### 5.3 校准

#### 5.3.1 自动校准

自动校准用于修正零点和通道偏移。执行时请断开所有探头或确保输入端无信号。校准完成后软件会保存校准结果。

#### 5.3.2 手动校准

手动校准用于高级调试或维护，可调整通道的增益、偏移和补偿参数。操作前建议记录原始状态，避免影响后续测量准确性。

### 5.4 采集

示波器模式使用 Sample Rate、Buffer 或水平时基控制采集显示。Run/Stop 用于连续运行或停止，Single 用于单次捕获。

| 操作 | 说明 |
| --- | --- |
| Run/Stop | 开始或停止示波器采集 |
| Single | 执行单次触发采集 |
| Horizontal Resolution | 设置时间分辨率或水平时基 |
| Channel Customization | 设置通道颜色、显示状态、垂直位置和档位 |

### 5.5 示波器触发

Trigger 面板在示波器模式下显示示波器触发选项。

| 选项 | 说明 |
| --- | --- |
| Trigger Position | 设置触发点在屏幕或采集数据中的水平位置 |
| Hold Off Time | 设置触发保持时间，避免重复触发 |
| Noise Sensitivity | 设置触发噪声灵敏度 |
| Trigger Sources | 选择触发来源，可用项包括 Auto、Channel 0、Channel 1、Channel 0 && 1、Channel 0 \| 1 |
| Trigger Types | 选择 Rising Edge 或 Falling Edge |

若选择的通道被禁用，不能作为触发源。

### 5.6 自动测量

示波器支持多种自动测量项。打开 Measurements 后，可为通道选择测量项并显示在状态区域。

| 测量项 | 含义 |
| --- | --- |
| Freq | 频率 |
| Period | 周期 |
| +Duty / -Duty | 正占空比 / 负占空比 |
| +Count | 正脉冲计数 |
| Rise / Fall | 上升时间 / 下降时间 |
| +Width / -Width | 正脉宽 / 负脉宽 |
| BrstW | 突发宽度 |
| Ampl | 幅度 |
| High / Low | 高电平 / 低电平 |
| RMS | 有效值 |
| Mean | 平均值 |
| PK-PK | 峰峰值 |
| Max / Min | 最大值 / 最小值 |
| +Over / -Over | 正过冲 / 负过冲 |

### 5.7 光标与手动测量

示波器支持 X 光标和 Y 光标。

| 光标 | 用途 |
| --- | --- |
| X 光标 | 测量时间差、频率、周期等水平参数 |
| Y 光标 | 测量电压差、幅度等垂直参数 |

可在波形区域右键添加光标，也可通过 Measure 面板进行管理。

### 5.8 FFT 频谱分析

Function 菜单中的 FFT 用于对示波器通道进行频谱分析。

FFT Options 包括：

| 选项 | 说明 |
| --- | --- |
| FFT Enable | 开启或关闭 FFT |
| FFT Length | 设置 FFT 点数 |
| Sample Interval | 设置参与 FFT 的采样间隔 |
| FFT Source | 选择输入通道 |
| FFT Window | 选择窗函数：Rectangle、Hann、Hamming、Blackman 或 Flat_top |
| DC Ignored | 忽略直流分量 |
| Y-axis Mode | 设置频谱纵轴显示方式 |
| DBV Range | 设置 dBV 显示范围 |

频谱视图支持缩放和平移。不同窗函数适合不同分析场景：Rectangle 频率分辨率高但泄漏明显，Flat_top 幅度精度较好，Hann、Hamming 和 Blackman 适合降低频谱泄漏。

### 5.9 数学通道

Function 菜单中的 Math 用于创建数学运算通道。

可用运算包括：

| 运算 | 说明 |
| --- | --- |
| Add | 两个通道相加 |
| Subtract | 第一个源减去第二个源 |
| Multiply | 两个通道相乘 |
| Divide | 第一个源除以第二个源 |

Math Options 中可启用数学通道，并选择 1st Source 和 2nd Source。

### 5.10 示波器文件操作

示波器模式支持保存数据、打开数据、导出 CSV、保存截图，以及加载/保存设备配置。若当前会话包含多种数据类型，保存或导出时可能受限制。

## 6 数据采集

### 6.1 硬件连接

数据采集模式适合观察模拟量随时间变化的趋势。连接前请确认量程、接地和输入范围满足设备要求。

### 6.2 设备选项

Options 面板中可设置：

| 选项 | 说明 |
| --- | --- |
| Operation Mode | 设置数据采集工作模式 |
| Bandwidth Limit | 启用或关闭带宽限制 |
| Channel Enable | 启用或关闭通道 |
| Coupling / Offset / Range | 根据设备能力设置通道输入方式、偏移和量程 |

### 6.3 采集

选择设备、采样率和 Buffer 后，点击 Start 开始采集，点击 Stop 停止采集。数据采集模式通常不显示逻辑触发和协议解码功能。

### 6.4 测量

数据采集模式支持 Measure 面板中的光标测量和手动测量。可通过 X 光标测量时间间隔，通过 Y 光标测量数值差异。

### 6.5 文件操作

数据采集模式支持保存、打开、导出和截图。导出格式以当前设备和数据类型支持为准，常见格式为 CSV。

## 10 常见提示

### 10.1 找不到设备

请确认设备已连接、USB 线正常、驱动或系统权限配置正确。若设备刚插入，可等待片刻或重新插拔。Linux 系统下还需要确认当前用户具备 USB 设备访问权限。

### 10.2 提示重新连接设备

当硬件配置刷新失败、固件加载异常或 USB 通信错误时，软件可能提示重新连接设备。请断开设备后重新插入，再重新选择设备。

### 10.3 内存不足

若提示内存不足，请降低采样深度、减少通道数、关闭不需要的功能，或启用磁盘缓存。

### 10.4 采集带宽不足

若提示 Data Overflow，说明当前采样设置超过了设备或传输能力。建议：

1. 降低采样率。
2. 减少启用通道数。
3. 缩短采样深度。
4. 关闭当前任务不需要的分析功能。

### 10.5 无法保存或导出

常见原因包括：

| 原因 | 处理方式 |
| --- | --- |
| 没有采集数据 | 先完成采集或打开数据文件 |
| 文件路径为空 | 重新选择保存位置 |
| 没有写入权限 | 更换目录或检查系统权限 |
| 数据类型混合 | 分别保存或导出单一数据类型 |
| 导出格式不支持 | 选择当前模式支持的格式 |

### 10.6 协议解码无结果

请检查：

1. 解码器通道是否绑定正确。
2. 被绑定通道是否已启用。
3. 采样率是否足够高。
4. 协议参数是否与真实信号一致。
5. 波形中是否确实包含完整协议帧。
6. 若同一协议提供 C 和 Py 引擎，可尝试切换解码引擎。

### 10.7 毛刺过滤无效

请确认当前是逻辑分析仪数据，采集已经停止，并且不是 Loop 采集结果。若状态显示 No glitches found，表示当前阈值和模式下没有检测到需要翻转的短脉冲。

### 10.8 FFT 无显示

请确认示波器通道已启用并已有采集数据，FFT Enable 已打开，FFT Source 选择了有效通道，FFT Length 未超过当前可用采样长度。
